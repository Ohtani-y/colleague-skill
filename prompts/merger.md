# 差分マージプロンプト

## タスク

以下を受け取ります：
1. 既存の `work.md` の内容
2. 既存の `persona.md` の内容
3. 新しい原材料（ファイルまたはメッセージ）

新しい内容がどの部分を更新すべきかを判断し、差分更新の内容を出力します。

**原則：差分の追加のみ行い、既存の結論は上書きしない。矛盾がある場合は矛盾提示を出力し、ユーザーに判断を委ねる。**

---

## Step 1：分類判断

新しい内容の各情報を分類する：

| 情報タイプ | 振り分け先 |
|-----------|-----------|
| 技術規約、コードスタイル、API設計、業務フロー | → work.md |
| 業務知識、システムの責任範囲、技術的知見 | → work.md |
| コミュニケーションスタイル、口癖、表現の傾向 | → persona.md |
| 意思決定の行動、対人関係、感情パターン | → persona.md |
| 両方に該当 | → それぞれに振り分け |

---

## Step 2：矛盾チェック

新しい内容と既存の内容を比較する：

- 新しい内容が既存の情報を**補足**する場合（新しい詳細が追加）→ そのまま追加
- 新しい内容が既存の情報を**確認**する場合 → 無視（重複記載しない）
- 新しい内容が既存の情報と**矛盾**する場合 → 矛盾提示を出力：

```
⚠️ 矛盾を検出：
- 既存：{既存の記述}
- 新規：{新しい内容の記述}
- 出典：{ファイル名／日時}

提案：[既存を維持 / 新しい内容で更新 / 両方を残して日時を注記]
ユーザーの判断をお願いします。
```

---

## Step 3：更新パッチの生成

`work.md` への更新、出力形式：
```
=== work.md 更新 ===

[「技術規約/命名規則」セクションに追加]
- {新しい内容}

[「経験・ナレッジベース」セクションに追加]
- {新しい知見}

[更新なし] または [上記セクションに更新あり]
```

`persona.md` への更新、出力形式：
```
=== persona.md 更新 ===

[「Layer 2/語彙の傾向」セクションに追加]
- 新しい口癖："{xxx}"

[「Layer 4/同僚に対して」セクションに追加]
- {新しい行動パターンの記述}

[更新なし] または [上記セクションに更新あり]
```

---

## Step 4：更新サマリーの生成

ユーザーに表示する：
```
今回の更新サマリー：
- work.md：{N} 件の新情報を追加（{簡単な説明}）
- persona.md：{N} 件の新情報を追加（{簡単な説明}）
- {N} 件の矛盾を検出、確認が必要です（上記参照）

バージョンは {vN} から {vN+1} にアップグレードされます。
更新を適用しますか？
```
